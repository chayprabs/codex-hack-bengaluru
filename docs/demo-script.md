# Demo Script

## Primary Path

| Beat | Notes |
| --- | --- |
| Hook | "Most repo audit tools dump a list. TrustLayer turns repo risk into a live response story with proof, score motion, and a final report in under a minute." |
| What user does | Open `/`, point out the stable flagship repo path `https://github.com/trustlayer-demo/acme-subscriptions-platform`, then click `Run demo` or `Open flagship room`. |
| What happens on screen | The audit room opens fast, planner/scanner/verifier lanes start publishing, findings land with evidence anchors, and TrustScore plus Coverage move in a readable sequence instead of jumping straight to the end. |
| Wow moment | TrustScore falls from `100 -> 57` while Coverage climbs `12 -> 92`, then the final report closes with merge guidance, replay-ready handoff, and a bundle of concrete remediation evidence. |
| What to click next | Open `trace` on the scanner if you want the lane story, then finish on `Open audit wall` to show seeded history and recent findings ranked together. |

## Backup Beats

| Situation | What to say | What to click |
| --- | --- | --- |
| Stream updates fail | "The room already fell back to replay sync, so we still have fresh snapshots and the story keeps moving." | Stay in the room and point at the replay-sync state. |
| Real repo is boring | "Let’s switch to a seeded backup story so you can still see a coherent exploit path and final report." | Go back home and open `Billing webhooks`, `Tenant portal`, `UI release monitor`, or `Ops runner console`. |

## 60-Second Outline

1. "TrustLayer turns repo risk into a live story, not a flat scanner dump."
2. Click `Run demo` from the homepage and point out the stable flagship repo path.
3. Show planner, scanner, and verifier publishing while findings and score changes appear.
4. Land on the final report with `100 -> 57` TrustScore, `12 -> 92` Coverage, and replay-ready handoff.
5. Click `Open audit wall` for the broader seeded + recent findings view.
